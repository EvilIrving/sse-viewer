# SSE Viewer - SSE 消息专用调试面板

## 🎯 解决什么问题？

Chrome Network 面板**可以看到** SSE 消息，但调试体验很糟糕：

## ✨ 对比：SSE Viewer vs Network 面板

| 操作           | Network 面板                      | SSE Viewer                 |
| -------------- | --------------------------------- | -------------------------- |
| 找到 SSE 请求  | ⚠️ 在一堆请求中翻找               | ✅ 自动汇总到专门面板      |
| 查看消息内容   | ⚠️ 逐个点击 EventStream 标签      | ✅ 直接显示所有消息        |
| 查看 JSON 数据 | ❌ 手动复制 → 粘贴到工具 → 格式化 | ✅ 点击自动展示格式化 JSON |
| 区分多个连接   | ❌ 混在一起难以区分               | ✅ 按连接自动分组          |
| 搜索历史消息   | ❌ 不支持                         | ✅ 关键词搜索              |
| 回看历史       | ⚠️ 刷新后丢失                     | ✅ 完整保留                |

## 🚀 使用方法

### 使用

1. **打开 DevTools** - 按 F12 或右键「检查」
2. **切换到 SSE Viewer 标签** - 在 Network、Console 旁边
3. **刷新页面** - 面板自动捕获所有 SSE 消息

## 🔒 隐私和安全

### 权限说明

扩展仅请求以下 3 个最小必要权限：

- 1. `activeTab` - 活动标签页访问
- 2. `scripting` - 脚本注入
- 3. `storage` - 本地存储

### 数据流向

```
页面 SSE 数据 → 拦截脚本 → DevTools 面板显示
     ↓
  仅本地处理，不上传任何数据
```

所有 SSE 消息数据：

- ✅ 仅在浏览器内存中处理
- ✅ 不写入磁盘（除非你手动导出）
- ✅ 不发送到任何外部服务器
- ✅ 关闭 DevTools 或刷新页面后自动清空
